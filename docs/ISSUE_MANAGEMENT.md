# 📋 问题管理工作流程

## 🎯 目标

建立标准化的问题记录、跟踪和解决流程，确保每个问题都有完整的文档记录。

## 📁 文件组织结构

```
docs/
├── issues/                          # 问题记录目录
│   ├── README.md                    # 问题索引和概览
│   ├── issue-XXX-brief-name/        # 具体问题文件夹
│   │   ├── README.md                # 问题详细描述
│   │   ├── files/                   # 相关文件
│   │   ├── logs/                    # 错误日志
│   │   └── solution/                # 解决方案文件
│   └── templates/                   # 问题模板
└── ISSUE_MANAGEMENT.md              # 本文件
```

## 🔢 问题编号规则

- **格式**: `issue-XXX-brief-name`
- **编号**: 三位数字，从001开始
- **名称**: 简短的英文描述，使用连字符分隔
- **示例**: `issue-001-dependencies`, `issue-002-github-upload`

## 📝 问题记录流程

### 1. 创建问题文件夹

```bash
mkdir -p docs/issues/issue-XXX-brief-name/{files,logs,solution}
```

### 2. 填写问题描述

使用模板创建 `README.md`，包含：
- 问题概述
- 详细描述
- 问题分析
- 解决方案
- 相关文件
- 测试验证
- 经验总结

### 3. 收集相关文件

- **files/**: 新创建或修改的文件
- **logs/**: 错误日志、调试信息
- **solution/**: 解决方案的具体实现

### 4. 更新问题索引

在 `docs/issues/README.md` 中添加新问题的记录。

## 🏷️ 问题分类标准

### 按类型分类
- **🔧 环境配置**: 安装、依赖、环境问题
- **🐛 Bug修复**: 代码错误、功能异常
- **💡 功能增强**: 新功能、改进建议
- **📤 部署上传**: 发布、部署相关问题
- **📚 文档更新**: 文档错误、缺失
- **🔒 安全问题**: 权限、安全相关

### 按严重程度分类
- **🔴 严重**: 阻塞性问题，影响核心功能
- **🟡 中等**: 影响用户体验，有解决方案
- **🟢 轻微**: 小问题，不影响主要功能

### 按状态分类
- **🔄 进行中**: 正在解决
- **✅ 已解决**: 问题已完全解决
- **⏸️ 暂停**: 暂时搁置
- **❌ 无法解决**: 技术限制无法解决

## 🔄 工作流程

### 遇到新问题时

1. **立即记录**: 创建问题文件夹和基本描述
2. **收集信息**: 保存错误日志、相关文件
3. **分析问题**: 找出根本原因
4. **制定方案**: 设计解决方案
5. **实施解决**: 执行解决方案
6. **验证测试**: 确认问题解决
7. **文档更新**: 完善问题记录
8. **经验总结**: 提取可复用的经验

### 定期维护

- **每周回顾**: 检查未解决问题的进展
- **每月整理**: 归档已解决问题，更新索引
- **季度总结**: 分析问题趋势，改进流程

## 🔧 工具和脚本

### 创建问题脚本

```bash
#!/bin/bash
# create_issue.sh
ISSUE_NUM=$1
ISSUE_NAME=$2
ISSUE_DIR="docs/issues/issue-$(printf "%03d" $ISSUE_NUM)-$ISSUE_NAME"

mkdir -p "$ISSUE_DIR"/{files,logs,solution}
cp docs/templates/issue_template.md "$ISSUE_DIR/README.md"

echo "Created issue directory: $ISSUE_DIR"
```

### 问题统计脚本

```bash
#!/bin/bash
# count_issues.sh
echo "问题统计:"
echo "总数: $(ls docs/issues/issue-* | wc -l)"
echo "已解决: $(grep -l "✅ 已解决" docs/issues/issue-*/README.md | wc -l)"
echo "进行中: $(grep -l "🔄 进行中" docs/issues/issue-*/README.md | wc -l)"
```
